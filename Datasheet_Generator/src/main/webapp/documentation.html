Owl Overview

Owl is a tool for automatically generating stylized datasheets [1]. Owl uses configuration files and LaTeX style files to create automatically typeset datasheets (see Guides for example Owl files). Data for datasheets can either be entered manually in the Owl user interface or Owl can use a configuration file to search existing data sources.

This version of Owl is open-source and the source code for the assembly algorithms is available on github. If you are interested in having Owl tailored for your company or have interest in other design-automation software, please contact Lattice Automation.

Have additional questions or wish to provide feedback to the developers? Please post to the Owl Forum or contact the developers.

PDF Generation

Owl uses configuration files to determine the fields and organization of the datasheets. These files have a convention for specifying the type of data to be uploaded (see guides for this convention). Based upon the uploaded configuration, fields are generated for data entry in the user interface.

Once all data is entered, Owl uses runs LaTeX to produce typeset datasheets. Optional .sty files may also be used to further stylize the sheets.

Additional cloning methods are not currently supported, however, Owl's core algorithm is highly generalizable and new methods can be added with relative ease. Please contact the developers if you are interested in Owl support for an assembly method that is not currently supported.

Import Existing Data

Owl configuration files can also be used to search existing data sources. Owl will search existing sources for all fields in the input configuration file. If these fields are not found or do not comply with the expected data type, they are left empty.

This search currently only supports searches on the Standard Registry of Parts. Mappings to additional data sources such as JBEI ICE is in progress. Please contact the developers if you are interested in Owl support for additional data sources.

FAQ

What do I need to use Owl?
A user must provide a configuration file to specify which data fields will be in the datasheet. Example configuration files can be found here. LaTeX style files are optional input - if one is not provided, the default styling will be used.
What do I input into Owl?
An Owl configuration file is required and a LaTeX .sty file is optional. Details about this file can be found here. Example input files can be found here
What does Owl output?
Owl outputs a typeset, stylized PDF.
